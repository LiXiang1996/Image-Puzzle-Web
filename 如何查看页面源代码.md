# 如何查看页面源代码确认验证标签

## 🎯 目标

确认 Google 验证标签 `<meta name="google-site-verification" content="你的验证码" />` 已正确添加到网站的 `<head>` 部分。

---

## 📋 方法一：浏览器查看源代码（最简单）

### Chrome / Edge / Brave 浏览器

1. **打开你的网站**
   - 访问：`https://image-puzzle-web-skmv.vercel.app`

2. **打开开发者工具**
   - **方法1**：按快捷键 `F12` 或 `Ctrl + Shift + I`（Windows/Linux）或 `Cmd + Option + I`（Mac）
   - **方法2**：右键点击页面空白处，选择 **"检查"** 或 **"Inspect"**

3. **查看页面源代码**
   - **方法1**：在开发者工具中，点击顶部的 **"Elements"**（元素）标签
   - **方法2**：右键点击页面空白处，选择 **"查看网页源代码"** 或 **"View Page Source"**
   - **方法3**：按快捷键 `Ctrl + U`（Windows/Linux）或 `Cmd + Option + U`（Mac）

4. **查找验证标签**
   - 在源代码中，按 `Ctrl + F`（Windows/Linux）或 `Cmd + F`（Mac）打开搜索框
   - 输入：`google-site-verification`
   - 应该能找到类似这样的代码：
     ```html
     <meta name="google-site-verification" content="abc123xyz456789" />
     ```

### Firefox 浏览器

1. **打开你的网站**
   - 访问：`https://image-puzzle-web-skmv.vercel.app`

2. **查看页面源代码**
   - **方法1**：按快捷键 `Ctrl + U`（Windows/Linux）或 `Cmd + Option + U`（Mac）
   - **方法2**：右键点击页面空白处，选择 **"查看页面源代码"**

3. **查找验证标签**
   - 在源代码中，按 `Ctrl + F`（Windows/Linux）或 `Cmd + F`（Mac）打开搜索框
   - 输入：`google-site-verification`
   - 应该能找到验证标签

### Safari 浏览器（Mac）

1. **打开你的网站**
   - 访问：`https://image-puzzle-web-skmv.vercel.app`

2. **启用开发者菜单**（如果还没有）
   - 打开 Safari 设置 → 高级 → 勾选"在菜单栏中显示'开发'菜单"

3. **查看页面源代码**
   - 点击顶部菜单栏的 **"开发"** → **"显示页面源代码"**

## 📋 方法二：使用浏览器开发者工具（Elements 标签）

这个方法可以看到浏览器实际渲染的 HTML，包括 Vue 应用渲染后的内容。

### 步骤

1. **打开你的网站**
   - 访问：`https://image-puzzle-web-skmv.vercel.app`

2. **打开开发者工具**
   - 按 `F12` 或右键选择 **"检查"**

3. **切换到 Elements 标签**
   - 在开发者工具顶部，点击 **"Elements"**（元素）标签

4. **展开 `<head>` 标签**
   - 在 Elements 面板中，找到 `<head>` 标签

5. **查找验证标签**
   - 在 `<head>` 标签内，查找 `<meta>` 标签
   - 找到 `name="google-site-verification"` 的 meta 标签
   - 应该能看到类似这样的代码：
     ```html
     <meta name="google-site-verification" content="abc123xyz456789">
     ```

## 🔍 如何确认验证标签正确

### 正确的验证标签格式

```html
<meta name="google-site-verification" content="你的验证码" />
```

**检查要点**：
- ✅ `name="google-site-verification"` 必须完全匹配
- ✅ `content` 属性包含你的验证码（Google 给你的那一串字符）
- ✅ 标签在 `<head>` 标签内
- ✅ 标签没有被注释（不是 `<!-- ... -->`）

### 错误的示例

```html
<!-- 错误：被注释了 -->
<!-- <meta name="google-site-verification" content="abc123" /> -->

<!-- 错误：name 属性错误 -->
<meta name="google-verification" content="abc123" />

<!-- 错误：content 属性为空 -->
<meta name="google-site-verification" content="" />
```

## 🐛 常见问题

### Q1: 找不到验证标签怎么办？

**可能原因**：
1. 代码还没有部署
   - **解决**：等待 Vercel 部署完成（通常1-2分钟）
   - 检查 Vercel Dashboard 确认部署状态

2. 验证标签被注释了
   - **解决**：检查 `index.html`，确保标签没有被 `<!-- -->` 注释

3. 浏览器缓存
   - **解决**：按 `Ctrl + Shift + R`（Windows/Linux）或 `Cmd + Shift + R`（Mac）强制刷新
   - 或者清除浏览器缓存

4. 查看的是错误的页面
   - **解决**：确保访问的是首页 `https://image-puzzle-web-skmv.vercel.app/`（不是 `/home` 或其他路由）

### Q2: 验证标签在源代码中，但 Google 验证失败？

**可能原因**：
1. Vercel CDN 缓存
   - **解决**：等待几分钟，CDN 可能需要时间更新

2. 验证码错误
   - **解决**：检查 `content` 属性中的验证码是否正确

3. 标签位置错误
   - **解决**：确保标签在 `<head>` 标签内，不是 `<body>`

### Q3: 如何确认代码已部署？

**方法**：
1. 检查 Vercel Dashboard
   - 访问：https://vercel.com/dashboard
   - 查看最新部署状态，应该显示 "Ready"（就绪）

2. 检查部署时间
   - 在 Vercel Dashboard 中，查看最新部署的时间
   - 如果刚刚提交代码，等待1-2分钟

3. 强制刷新页面
   - 按 `Ctrl + Shift + R` 强制刷新，绕过浏览器缓存
